Task 2 — Logical and Physical Models for Restaurant Data

 

 

Description

Your friend is starting a restaurant business. She has ambitions to add franchise locations, and food trucks. She is "data-hungry" and wants to use data to support her operations / inform her strategy. Create a data model for her company (you can make up the details). At minimum, please include a logical and a physical model (you get to make up the tables, relationships, keys, etc).

 

Submission


credit: https://vertabelo.com/blog/a-restaurant-delivery-data-model/

placed_order


CREATE TABLE placed_order (
  id INTEGER,
  restaurant_id INTEGER,
  order_time timestamp,
  estimated_delivery_time timestamp,
  food_ready timestamp,
  actual_delivery_time timestamp,
  delivery_address varchar(255),
  customer_id integer,
  price decimal(12,2),
  discount decimal(12,2),
  final_price decimal(12,2),
  comment varchar(255),
  discount timestamp,
  PRIMARY KEY(id)
)

 

in_order


CREATE TABLE in_order (
  id INTEGER,
  placed_order_id INTEGER,
  offer_id INTEGER,
  menu_item_id INTEGER,
  quantity INTEGER,
  item_price DECIMAL(12,2),
  price DECIMAL(12,2),
  comment VARCHAR(255),
  PRIMARY KEY(id)
)

 

offer


CREATE TABLE in_order (
  id INTEGER,
  date_active_from DATE,
  time_active_from DATETIME,
  date_active_to DATETIME,
  time_active_to DATETIME,
  offer_price DECIMAL(12,2),
  PRIMARY KEY(id)
)
